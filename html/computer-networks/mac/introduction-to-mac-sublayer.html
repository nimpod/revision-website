Introduction to the MAC Sublayer

Layer 2.1 - The MAC sublayer is responsible for moving data packets to and from one Network Interface Card (NIC) to another across a shared channel

MAC = Medium Access Control

The MAC sublayer is one of two sublayers that make up the Data Link Layer of the OSI model. There is so much content to cover in MAC that I thought it deserved it's own section.



What is the difference between the MAC and LLC sublayers?

  • MAC - moves data across a shared channel and does part of the framing.
  • LLC - connects different LAN's together. Does error handling, flow control and does part of the framing.

It might not make any sense to you right now, but the MAC sublayer uses the 1-persistent CSMA/CD protocol with the Binary Exponential Backoff Protocol.

  • 1-persistent: stations send as soon as link is idle
  • CSMA/CD: stations can detect collisions whilst transmitting
  • Binary Exponential Backoff: used to decide how long to wait until next retransmission

Channel Allocation Problem

Network links can be PTP or broadcast.

In broadcast networks, there is a single shared transmission medium in which everyone communicates via.

Adding new hosts to the network is easy. (eg. quickly connecting to your home WiFi, quickly connecting to the uni WiFi , etc...)

The Channel Allocation Problem asks 'how do you allocate a single broadcast channel amongst other competing users?'.

There are two solutions to the problem.

  1. Static Channel Allocation - if (traffic is quiet and predictable) -> channel is split equally among users.
  2. Dynamic Channel Allocation - if (traffic is busy and unpredictable) -> use Multiple Access Protocols.
