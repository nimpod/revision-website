Circuit-Switch and Packet-Switch

The purpose of a computer network is to provide services to users (that sounds like a very vague thing to say, but I will try to make it clearer...)

Those services may need...

  • Guarnteed bandwidth -> Circuit-Switch
  • Expected bandwidth -> Packet-Switch

Circuit-Switch

  • Dedicated point-to-point link between 2 devices.
  • You have to setup a path before you can start talking.

Packet-Switch

  • Allows many devices to use the same link.
  • Before the message is sent, the message is broken down into chunks (aka. packets).
  • There are 2 services in Packet-Switch:
    1. Virtual circuit - setup a circuit. Sender and receiver communicate directly (connection orientated).
    2. Datagram - don't setup a circuit. Messages are sent immediatley (connectionless).

Connection-Orientated vs Connectionless

Connection-Orientated:

  • Route chosen when virtual circuit is set up; all packets follow it.
  • Packets arrive in order.
  • Each packet contains a short vc_id instead of its destination address.
    • Examples of connection-orientated services
    Service Example
    Reliable message stream Sequence of pages
    Reliable byte stream Movie download
    Unreliable connection Video/voice call
    Reliable connection File transfer

    What exactly do I mean by an 'unreliable' connection?

    • No guarantee that all packets will arrive in order.
    • No guarantee that a packet will not get lost.
    • No guarantee that a packet will not get corrupted.
    • No guarantee that a packet will go to the wrong destination.

    What is the difference between Message stream and Byte stream?

    • Message stream reserves boundaries.
    • Byte stream does not reserve boundaries.
    • Message stream is about sending a continuous flow of data.
    • Byte steam is about sending data in distinct chunks or groups

Connectionless:

  • Each packet is routed independently of its predecessors. Successive packets may follow different routes.
  • Packets may arrive in a different order from that in which they were sent.
  • Each datagram must contain the full destination address.
    • Examples of connectionless services
    Service Example
    Unreliable datagram Electronic junk email
    Reliable datagram Email
    Acknowledged datagram Text messaging
    Request-reply Database query
