OSI Model

The Open Systems Interconnection (OSI) model is a conceptual model that characterises and standardises the communication functions of a telecommunication or computing system.

It conceptually divides computer network architecture into 7 layers.

  • The higher layers (7,6,5,4) cover network requests/responses, and representing data in meaningful ways.
  • Lower layers (3,2,1) deal with electrical signals, binary data and routing data across networks.

As you travel down the OSI model, you add headers to each layer.

There are 7 OSI layers:

Each OSI layer's responsibilities and functions
Level Name Description Protocol Data Unit
7 Application Some application designed to be used by an end-user (e.g: SSH, FTP, SMTP, HTTP(S), DNS)
6 Presentation Irrelevant to this module
5 Session Irrelevant to this module
4 Transport Decides how much data will be passed through. Encapsulates packets into segments. This is about security and validity. Scans the data to check for errors. Capable of error recovery, flow control, and re-transmission. Data that is not recovered from an error will not pass through, and may be destroyed. Segment
3 Network Encapsulates the data into separate packets. Also determines the best, most reliable, fastest route to deliver some data. (Routers are involved in this layer). The network layer also adds a source and destination IP address to each packet. Packet
2 Data Link Switches operate on this layer. Packets are turned into frames, and error-free data transfer of the frames is provided, over into the Physical Layer. MAC addresses! Frame
1 Physical Responsible for the transmission of digital data bits. Uses physical medium like copper cable, fibre optic cable, WiFi, Ethernet or infrared. Bit

Network engineers use the OSI model as a conceptual tool to classify problems. For example...

  • If your computer is not connected to the network => Application Layer problem
  • If you put in the wrong IP address => Network Layer problem
  • If the router dies => Network Layer problem
  • If a cable gets cut, or is mis-wired => Physical Layer problem

The remainder of this module is structured around the OSI model, using it as a framework to teach important concepts about computer networks.
