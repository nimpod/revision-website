Cyclic Redundancy Check (CRC)

This is another method of error detection.

It is kind of like a refined version of checksum, with stronger error detection.

Computing the CRC requires XOR long division. Firstly Let's just do a super basic example of division (no generators yet):

Example of XOR long division:
---------
Calculate 101/100100 
101 | 100100
      101
      ---
      00110
        101
        ---
        0110
         101
         ---
         011
∴ final answer = 011

Unfortunatly for us, CRC isn't that simple. We need to do a few extra steps before and after the division:

Example of computing the CRC:
---------
A bit stream of 10011101 is transmitted using the standard CRC method. The generator is 1001. Compute the CRC:

1) know what the generator and data are: 
    generator = 1001
    data = 10011101
2) Boundary case: (len(generator) < len(data)) must be True in order to proceed with CRC. 
    len(generator) = 4, len(data) = 8:
    4 < 8 = True 
    ∴ we can continue
3) Append len(generator)-1 zero's to the end of data 
    len(generator)-1 = 3
    data = 10011101000
4) Do the XOR division 
1001 | 10011101000
       1001
       ----
       00001101
           1001
           ----
           01000
            1001
            ----
            000100
5) Finally, sometimes we want to find the actual bit string that is transmitted. To do this you simply do 'data + CRC' 
transmitted bit string = 10011101000 + 100
transmitted bit string = 10011101100

IEEE 802 notation

In the exam they don't give us the generator in binary.

No no no, they give it to us in flippin' algebra just to piss us off.

It's pretty easy to convert it though. Here are some examples:

111 = x2 + x1 + x0
001 = x0
101101 = x5 + x3 + x2 + x0
1001 = x3 + x0

Examples

Example 1
---------
A bit stream 10101110 is transmitted along with a CRC calculated using the generator polynomial: x3 + x2 + x0
a) What is the CRC value?
b) What is the actual bit string that gets transmitted? 
a)
    x3 + x2 + x0 = 1101
    ∴ generator = 1101
    ∴ data = 10101110000

    1101 | 10101110000
           1101
           ----
           01111
            1101
            ----
            001011
              1101
              ----
              01100
               1101
               ----
               0001
                  1000
                  1101
                  ----
                  0101

    ∴ CRC value = 101
b)
    transmitted bit string = 10101110000 + 101
    transmitted bit string = 10101110101
